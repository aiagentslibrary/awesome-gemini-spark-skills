# Awesome Gemini Spark Skills

**A curated library of copy-ready Gemini Spark skills, Google Workspace AI agent workflows, and connected-app automation templates.**

Use these `SKILL.md` files to create practical Gemini Spark workflows for Gmail, Calendar, Drive, Docs, Sheets, Slides, YouTube, Maps, Canva, GitHub, and more.

## What This Repo Helps You Do

This repo is for people who want practical Gemini Spark workflows, not complicated AI agent theory.

You can use it to create skills for:

- Gmail inbox summaries and reply drafts
- Calendar planning and meeting prep
- Drive organization and file discovery
- Google Docs guides, SOPs, checklists, and reports
- Google Sheets trackers and dashboards
- Google Slides deck outlines
- YouTube research and content planning
- Canva creative briefs
- Google Photos asset discovery
- Google Maps trip planning
- GitHub repo analysis

---

## Quick Start

1. Open a skill folder.
2. Copy the `SKILL.md` content.
3. Paste it into your Gemini Spark skill setup area.
4. Connect only the apps needed for that skill.
5. Test with a small, low-risk task first.
6. Keep approval turned on for sensitive actions.

Recommended first skill:

```text
skills/start-here/spark-daily-brief/SKILL.md
```

## Approval-First Design

These skills are designed to help Spark draft, summarize, organize, and prepare work quickly while asking before sensitive actions.

| Approval Level | Meaning |
|---|---|
| Low | Summaries, drafts, plans, research, and recommendations |
| Medium | Creating or editing files, tasks, labels, rows, or calendar drafts |
| High | Sending messages, sharing files, deleting data, posting publicly, booking, buying, or changing permissions |

See [`docs/approval-levels.md`](docs/approval-levels.md) for the approval model.

## Skill Format

Every skill follows a simple structure:

```text
---
name: skill-name
description: Clear description and trigger phrases.
---

# Skill Title

## Main Goal
## Works Best With
## Welcome Message
## Workflow
## Output Format
## Approval Rules
```

See [`docs/skill-template.md`](docs/skill-template.md) for a reusable template.
